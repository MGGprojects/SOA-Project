This frontend talks only to the API gateway. The gateway routes requests to the existing microservices for events, calendar exports, authentication, and protected business actions.

Gateway Health

Architecture Flow

  1. Frontend requests data from the gateway.
  2. Gateway routes `/api/auth` to `auth-service` for login and JWT validation.
  3. Gateway routes `/api/events` and `/api/businesses` to protected backend services.
  4. User can browse public data or log in to perform protected actions.
